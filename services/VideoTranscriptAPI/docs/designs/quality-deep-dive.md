## 愿景

将 VideoTranscriptAPI 从"能用的转录工具"升级为"可信赖的转录引擎" — 校对纠错率 95%+、生产环境零静默失败、完整的性能可见性。

## 实施阶段

### Phase 1: 基础设施 (SQL 修复 + WAL + 迁移系统)

**目标**：修复安全漏洞，提升数据层并发性能，建立 schema 变更基础。

**改动清单**：

1. **SQL 参数化修复** (`audit_logger.py`)
   - 文件：`src/video_transcript_api/utils/logging/audit_logger.py`
   - 行 175, 185, 197, 243：将 `.format(days)` 改为 SQLite 参数化
   - 注意：SQLite 的 `datetime('now', '-? days')` 不支持参数化，需要用 Python 计算日期后传入

2. **SQLite WAL 模式** (`cache_manager.py` + `audit_logger.py`)
   - 在 `_get_connection()` 和 `_init_database()` 中添加 `PRAGMA journal_mode=WAL`
   - **注意**：NFS 文件系统不支持 WAL，需要 try/except 降级：
     ```python
     try:
         conn.execute("PRAGMA journal_mode=WAL")
     except sqlite3.OperationalError:
         logger.warning("WAL mode not supported, using default journal mode")
     ```

3. **简单版本检查** (`cache_manager.py` + `audit_logger.py`)
   - 添加 `schema_version` 表（单行单列）
   - 在 `_init_database()` 中检查版本，按需运行升级 SQL
   - 格式：
     ```python
     CURRENT_SCHEMA_VERSION = 2

     def _check_and_migrate(self):
         version = self._get_schema_version()  # 无表则返回 0
         if version < 1:
             self._migrate_v1()  # 创建初始表
         if version < 2:
             self._migrate_v2()  # 新增字段
         self._set_schema_version(CURRENT_SCHEMA_VERSION)
     ```

4. **AuditLogger 连接复用**
   - 参照 CacheManager 的 `threading.local()` 模式
   - 去掉每次请求新建连接的开销

**测试**：
- `tests/unit/test_audit_logger.py` — SQL 参数化验证、连接复用
- `tests/unit/test_migration.py` — 版本检查、升级流程

### Phase 2: 错误体系 + God Class 拆分

**目标**：建立统一的错误分类体系，拆分 1658 行的 transcription.py。

**改动清单**：

1. **统一错误分类** (新建 `src/video_transcript_api/errors/`)
   ```
   errors/
   ├── __init__.py
   ├── base.py          # TranscriptAPIError 基类
   ├── network.py       # NetworkError, DownloadTimeoutError, HTTPForbiddenError
   ├── transcription.py # ASRConnectionError, EmptyTranscriptError
   └── download.py      # DownloadFailedError, InvalidMediaError
   ```
   - 参考 `llm/core/errors.py` 的 classify_error 模式
   - 所有下载器的 `except Exception` 改为具体异常类

2. **下载器重试逻辑** (`downloaders/base.py`)
   - 在 `download_file()` 中添加重试装饰器：
     ```python
     @retry(max_retries=3, retry_on=(NetworkError, DownloadTimeoutError))
     def download_file(self, url, filename):
         ...
     ```
   - 清理死代码（base.py:246-271）

3. **transcription.py 函数拆分**
   ```
   services/
   ├── transcription.py    # 保留: API 模型、verify_token、process_task_queue、process_llm_queue
   ├── download_ops.py     # 拆出: 下载相关逻辑
   ├── transcribe_ops.py   # 拆出: 转录相关逻辑
   └── llm_ops.py          # 拆出: _handle_llm_task 及 LLM 相关逻辑
   ```
   - transcription.py 变为薄胶水层，import 并调用各 ops 模块

**测试**：
- `tests/unit/test_error_classification.py` — 错误分类
- `tests/unit/test_download_retry.py` — 重试逻辑
- `tests/unit/test_download_ops.py` — 下载操作（mock 网络）

### Phase 3: 功能增强 (专词库 + 性能埋点 + 输入验证)

**目标**：提升校对质量，建立性能可见性，加固输入安全。

**改动清单**：

1. **专有名词库** (新建 `src/video_transcript_api/terminology/`)
   ```
   terminology/
   ├── __init__.py
   ├── terminology_db.py   # 加载、查询、注入 prompt
   └── data/
       └── default_terms.json  # 默认专词库
   ```
   - JSON 格式：
     ```json
     {
       "version": 1,
       "terms": [
         {"incorrect": "GPT 4", "correct": "GPT-4", "category": "tech"},
         {"incorrect": "闷课", "correct": "MOOC", "category": "edu"}
       ]
     }
     ```
   - 在 LLM prompt 中注入：`"以下是专有名词对照表，请在校对时使用正确写法：\n{terms}"`
   - 支持 `config/terms.json` 用户自定义路径

2. **性能埋点** (新建 `src/video_transcript_api/utils/perf_tracker.py`)
   - 使用上下文管理器记录各阶段耗时：
     ```python
     with perf_tracker.track("download", task_id=task_id):
         audio_path = download_ops.download(...)
     ```
   - 缓存命中/未命中计数
   - 日志输出 + 可选持久化到 SQLite

3. **输入验证加强** (`tasks.py` 或新建 `utils/validators.py`)
   - webhook URL：复用 `url_validator.validate_url_safe()` 逻辑
   - metadata 长度限制：title ≤ 200 字符，description ≤ 2000 字符

**测试**：
- `tests/unit/test_terminology.py` — 加载、查询、prompt 注入
- `tests/unit/test_perf_tracker.py` — 耗时记录、统计
- `tests/unit/test_input_validation.py` — webhook 验证、metadata 限制

### Phase 4: 运维 (健康检查 + ASR 告警)

**目标**：让系统状态可见，ASR 宕机时自动通知。

**改动清单**：

1. **健康检查端点** (`app.py` 或 `routes/health.py`)
   ```python
   @router.get("/health")
   async def health_check():
       checks = {
           "sqlite": check_sqlite(),       # SELECT 1
           "capswriter": check_capswriter(), # WebSocket ping
           "funasr": check_funasr(),        # WebSocket ping
           "disk_space": check_disk(),      # df / statvfs
       }
       status = "healthy" if all(checks.values()) else "degraded"
       return {"status": status, "checks": checks}
   ```
   - Docker compose 添加 healthcheck：
     ```yaml
     healthcheck:
       test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
       interval: 30s
       timeout: 10s
       retries: 3
     ```

2. **ASR 宕机告警** (在健康检查基础上)
   - 后台定时检查（每 5 分钟）
   - 连续 N 次失败时发送企微通知
   - 恢复后发送恢复通知
   - 防抖：告警后 30 分钟内不重复告警

**测试**：
- `tests/unit/test_health.py` — 各组件健康检查（mock WebSocket）
- `tests/unit/test_asr_alert.py` — 告警触发、防抖、恢复通知

## 关键架构决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| God Class 拆分策略 | 函数拆分（非管道模式） | 最小 diff，保持现有调用方式不变 |
| 迁移系统 | 简单版本检查（非 Alembic） | 项目规模小，单人开发，过度工程 |
| 专词库格式 | JSON 文件（非 SQLite） | 简单直接，可版本控制 |
| 测试时机 | 每 Phase 自带测试 | 测试不可商量，防止回归 |
| 分阶段实施 | 5 个 Phase（非一次性） | 降低爆炸半径，增量优于革命 |

## 已知风险

| 风险 | 缓解措施 |
|------|---------|
| WAL 在 NFS 上不支持 | try/except 降级到默认 journal |
| 健康检查可能误报 | ASR ping 应发送实际转录请求（小音频） |
| 函数拆分可能引入循环导入 | 延迟导入 + 仔细管理依赖方向 |
